# Локализация (i18n)

Бот и дашборд говорят на 10 языках. Язык выбирает пользователь в боте (`/start`
при первом запуске, `/settings` — в любой момент) или в дашборде (переключатель
в шапке). Выбор хранится в одном месте — `bot_users.language` — и единообразно
применяется к обоим UI.

## Поддерживаемые языки

| Код | Язык | Направление |
|---|---|---|
| `ru` | Русский | LTR |
| `en` | English | LTR (универсальный fallback) |
| `es` | Español | LTR |
| `pt-BR` | Português (Brasil) | LTR |
| `de` | Deutsch | LTR |
| `tr` | Türkçe | LTR |
| `id` | Bahasa Indonesia | LTR |
| `fa` | فارسی | **RTL** |
| `ar` | العربية | **RTL** |
| `zh` | 中文 (简体) | LTR |

Каноничный список — `com.tcleaner.core.BotLanguage`.

## Архитектура

```
┌──────────────┐           ┌──────────────┐           ┌──────────────┐
│  Бот /start  │──выбор──▶ │  BotUser     │ ◀─чтение──│  Дашборд     │
│  /settings   │           │  .language   │           │  Thymeleaf   │
└──────────────┘           └──────────────┘           └──────────────┘
       │                          ▲                          ▲
       └─ BotUserUpserter ────────┘         BotUserLocale-   │
          .setLanguage()                    Resolver читает  │
                                            через principal  │
                                            → botUserId      │
```

- **Бот** читает/пишет `language` через `BotUserUpserter` (см.
  `com.tcleaner.dashboard.service.ingestion.BotUserUpserter`).
- **Дашборд** читает через `BotUserLocaleResolver`, пишет через
  `POST /dashboard/api/me/settings/language`.

## Ресурсные файлы

Два независимых bundle-а — разные владельцы текстов, раздельные циклы обновления:

- `src/main/resources/bot_messages[_<lang>].properties` — тексты бота.
  Конфиг: `com.tcleaner.bot.BotI18nConfig` → bean `botMessageSource`.
- `src/main/resources/messages[_<lang>].properties` — тексты дашборда
  (использует стандартный Spring `MessageSource`, который подключается
  автоматически).

Базовый файл (`bot_messages.properties`, `messages.properties`) — **English**,
используется как fallback, если в нужной локали нет ключа.

Кодировка — UTF-8 (критично для `zh/ar/fa`).

## Как добавить новый язык

1. Добавить значение в `BotLanguage` (код, `displayName` с флагом, `Locale`,
   RTL-флаг).
2. Создать два файла: `bot_messages_<code>.properties` и
   `messages_<code>.properties`. Скопировать ключи из EN/RU версии, перевести.
3. Если язык RTL — добавить в `DashboardWebConfig`/`layout.html` условие для
   `dir="rtl"` (сейчас явная проверка `fa`/`ar`, при добавлении нового RTL —
   расширить список).
4. Тесты не требуют правок (набор резолвится из enum).
5. `setMyCommands` подхватит язык автоматически — см.
   `ExportBot.registerBotCommands`.

## Как добавить новый ключ

1. Добавить ключ в `bot_messages.properties` (EN, fallback) и
   `bot_messages_ru.properties`. Остальные локали подтянут EN до перевода —
   это приемлемая temporary-state.
2. В коде использовать `i18n.msg(lang, "key", args)`.
3. Постепенно добавить ключ в остальные 8 языков.

Аналогично для дашборда — начиная с `messages.properties` (EN) и
`messages_ru.properties`.

## Fallback-поведение

- Если у пользователя `language == null` — бот при `/start` показывает
  клавиатуру выбора. Дашборд (если вошёл) резолвит `Accept-Language` из
  браузера, иначе — `ru` (историческая база пользователей).
- Если ключ отсутствует в локали `X` — Spring автоматически берёт из
  `bot_messages.properties` / `messages.properties` (EN).
- Если ключ отсутствует везде — `BotI18n.msg` возвращает сам ключ (не падает).

## RTL

Для `fa` и `ar`:
- `layout.html` рендерит `<html dir="rtl">`.
- Подключается `/dashboard/css/rtl.css` с зеркальными стилями.
- Коды языков и числа в таблицах — LTR override.
