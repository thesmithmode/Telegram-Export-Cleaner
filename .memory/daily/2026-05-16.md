### Incident: prod 404 после dev→main merge

Root cause: `tecnativa/docker-socket-proxy:0.3.0` НЕ совместим с `read_only:true` и `tmpfs:/usr/local/etc/haproxy`.

- Образ при старте `sed`-ит `/usr/local/etc/haproxy/haproxy.cfg.template` → пишет `.cfg` в ту же папку.
- `read_only:true` блокирует запись → crash loop.
- `tmpfs:/usr/local/etc/haproxy` (попытка "починить") перекрывает шаблон image layer → `sed: ...template: No such file` → тоже crash.
- Каскад: socket-proxy down → Traefik docker-provider не подключается → нет роутов → **404 на ВСЁМ трафике**.

False-positive в CI: health check был `docker exec telegram-export-java-bot wget localhost:8080/api/health` — обходил Traefik. java-bot жив напрямую → CI зелёный → автоматического rollback нет → юзеры получали 404 пока инцидент не заметили.

Фиксы:
- `f435262` — добавил в `build.yml` диагностику Traefik + smoke test через сам Traefik (`curl -H "Host: $DOMAIN" http://localhost/dashboard/login`) с проверкой 200/301/302.
- `d6e4ef4` — убрал `read_only:true` и оба `tmpfs` из socket-proxy. Defense-in-depth остаётся: API-фильтры (CONTAINERS=1, POST=0, ...), изолированная docker-proxy сеть, SHA-pin образа.
- `4854a8f` — добавил предупреждающий коммент в `docker-compose.prod.yml` чтобы не повторить.
