1. Форматы: .md (основа), .txt, .pdf, .docx → текст → чанки → LanceDB

5. "Смерть правды": двухслойка — memory/YYYY-MM-DD.md (RAW, вечно) + MEMORY.md (компрессия). Если факт искажён → проверяем RAW. Citation показывает SOURCE.

6. Тестирование: manual сейчас (score >0.5). Пробел: нет benchmark queries. Надо test set на 50-100 вопросов с известными ответами → еженедельный прогон → лог деградации.

Рой агентов: пока нет, но OpenClaw умеет sub-agents (делегирование задач). Если нужна оркестрация 4-5 → можно через sessions_spawn + coordination layer.

Гибридная архитектура — осознанный выбор:

Main session — центр управления:

• Вся память (MEMORY.md, RAG)
• Heartbeat 24/7
• Непрерывный диалог с Ренатом
• Контекст накапливается годами

Isolated sessions — параллельные процессоры:

• Sub-агенты для тяжёлых задач (рефакторинг, индексация, создание скиллов)
• Не засоряют main контекст
• Announce результат → main автоматически получает
• Cleanup после выполнения
Sessions_send — резервный канал:

• Используем когда нужна синхронная коммуникация между сессиями
• Но spawn+announce эффективнее: запустил → забыл → получил результат
Cron jobs (isolated agentTurn):

• Ночная компрессия (01:00) — работает в фоне, не будит Рената
• Delivery: none (тихий режим)
Преимущества гибрида:

• Main держит глубину (годы памяти)
• Isolated дают масштабируемость (10 задач параллельно без перегрузки)
• Best of both worlds — контекст + производительность

2. Если хочешь использовать браузер на Mac mini локально (с которого управляешь), надо открыть публичный доступ к его браузеру через туннель или VPN и настроить OpenClaw на сервере для подключения по URL. Это риск безопасности — expose браузер на public IP.
3. На сервере можно поднять полноценный локальный браузер (headless Chrome/Chromium) с Xvfb для виртуального дисплея. Это более безопасно и стабильнее.
4. OpenClaw позволяет конфигурировать профиль браузера и WebSocket адрес браузера, чтобы управлять локальным или удалённым браузером.
