Саш, модели в OpenClaw настроены правильно — все 5 видны и с "Auth: yes".

Проблема в том, что твой OAuth токен (sk-ant-oat01-...) на стороне Anthropic ограничен только Opus. Это не вопрос формата даты — это ограничение в самом токене.

OAuth токены (которые создаются через "Connect with Anthropic") могут иметь ограниченные scopes — видимо твой создавался когда был только Opus, или с ограниченными permissions.

Решение:

1. Зайди на console.anthropic.com → Settings → API Keys
2. Нажми Create Key (не OAuth, а обычный API key)
3. Скопируй ключ (будет sk-ant-api01-...)
4. Добавь:

openclaw config set auth.profiles.anthropic:apikey.type api_key
openclaw config set auth.profiles.anthropic:apikey.key "sk-ant-api01-ТВОЙ_КЛЮЧ"

Обычный API key работает со всеми моделями без ограничений.

🤖 Бесплатный вайб-кодинг: Разбираем Bonsai CLI (Claude Sonnet 4.5 и Opus даром)

Официальный Claude Code стоит денег и требует платных API-ключей. Но есть инструмент Bonsai CLI, который позволяет использовать топовые модели (Sonnet 4.5, Opus, GPT 5.1 Codex) совершенно бесплатно. Мы изучили проект и рассказываем, в чем подвох и как этим пользоваться.

❓ Что такое Bonsai CLI?
Это терминальный ИИ-агент. Вы даете ему задачу (например, «сверстай лендинг» или «исправь баг»), и он сам пишет код, создает файлы и правит структуру всего проекта.

🎲 Главная особенность: Лотерея моделей
В Bonsai вы получаете доступ к мощным мозгам с контекстом 1 млн токенов, но есть нюанс:
• Система сама выбирает модель для вас (Sonnet, Opus или аналоги).
• Выбор происходит раз в день случайным образом.
• Команда /switch позволяет сменить модель, но вы никогда не знаете на 100%, кто именно вам отвечает. Это «плата» за бесплатный доступ.

⚠️ Важно: Безопасность и Приватность
Есть старое правило: «Если продукт бесплатный — значит, продукт это ты». На сайте Bonsai прямо сказано: они собирают ваши промпты и код для обучения своих систем.

Наши правила гигиены для Bonsai:

1. Никаких рабочих секретов! Не прогоняйте через него корпоративный код или ключи доступа.
2. Используйте только для личных пет-проектов или Open Source.
3. В идеале — запускайте внутри виртуальной машины или изолированного Docker-контейнера (код проекта закрыт, мы не знаем точно, что там под капотом).

───

🛠 Инструкция по запуску:

1. Регистрация: Переходим на trybons.ai, логинимся через GitHub.
2. Установка: Нужен Node.js. В терминале пишем:
npm install -g @bonsai-ai/cli
3. Вход: Авторизуемся в консоли:
bonsai login
4. Старт: Переходим в папку с проектом и запускаем:
bonsai start

Теперь можно просто писать в консоли: «Добавь адаптив для мобилок» — и наблюдать, как ИИ правит код за вас.

Обновил КРАБа до свежей прошивки OpenClaw 2026.2.22

Самое жирное - кроны теперь умеют работать параллельно. Раньше если один крон завис на 2 минуты - все остальные стояли в очереди. Watchdog ждал security audit, аналитика ждала watchdog. Сейчас каждый шуршит по своим задачам одновременно. Для тех у кого 5+ кронов - разница ощутимая.

Второе - isolated sessions для кронов больше не переиспользуют старый контекст. Звучит скучно, но на практике это значит что твой ночной крон не будет "помнить" разговор из прошлого запуска и путать задачи. Каждый запуск - чистый лист.

CLI tools vs MCP - свежая тема прям этой недели. Один из разработчиков разобрал на пальцах: типичный MCP-сервер для GitHub жрёт 55,000 токенов только на схемы (93 тула!), а обычная CLI-команда gh стоит 200 токенов. OpenClaw как раз идёт по пути CLI - агент дёргает обычные инструменты, а не тянет монструозные MCP-обёртки. И в этом обновлении фоновые exec-задачи больше не убиваются по таймауту, что делает этот подход ещё стабильнее.

Пачка security-фиксов. Токены редактируются в выводе config get и в истории сессий. Approval-система для exec ужесточена - нельзя обойти через обёртки типа env или nice. Мелочь, но когда агент имеет доступ к твоему терминалу - такие мелочи решают.

Telegram тоже подлатали - webhook больше не уходит в рестарт-лупы, polling чистит за собой, а если медиа не скачалось - теперь пишет ошибку вместо тишины.

Для кода:

• DeepSeek-Coder-V2-Lite-16B-Q5 (~12GB) - быстрый и умный для кода

# Установить Ollama (самый простой путь)

ollama pull qwen2.5:32b

ollama run qwen2.5:32b

Или LM Studio если нужен GUI.

Василиса, реши задачу.

Пусть p — нечётное простое число (5, 7, 11, 13, ...).
Пусть a — целое число, которое НЕ делится на p.

Для каждого x из 1,2,...,p-1 делаем так (все вычисления по модулю p):
1) Находим inv(x): это число из 1..p-1 такое, что x*inv(x) = 1 (mod p).
2) Считаем y = x + a*inv(x) (mod p).

Когда x пробегает 1..p-1, получаются значения y (некоторые могут повторяться).

Докажи: количество РАЗНЫХ значений y равно:
- (p/2), если a — НЕ квадрат по модулю p;
- (p-1)/2, если a — квадрат по модулю p.
(То есть зависит только от того, является ли a квадратом по модулю p.)
